# 꼼꼼욕실 홈페이지 운영 설정 가이드

이 사이트는 방문 요청을 Worker에서 실행하지 않는 완전 정적 자산으로 제공됩니다. 최신 시공 후기 3개는 빌드 시 네이버 블로그 RSS에서 `data/blog-posts.json`으로 동기화됩니다. 서비스·FAQ·채널 정보는 `data/site.ts`를 원본 데이터로 사용합니다.

## 1. 도메인과 기본 주소

1. 최종 도메인을 구매하고 호스팅 서비스에 연결합니다.
2. 배포 환경 변수 `NEXT_PUBLIC_SITE_URL`에 `https://`를 포함한 최종 주소를 입력합니다.
3. `app/page.tsx`의 구조화 데이터와 `public/llms.txt`에 들어 있는 임시 주소 `https://ggomggombath.com`을 실제 주소로 변경합니다.
4. HTTPS, `www` 사용 여부, 대표 도메인 리디렉션을 한 가지로 통일합니다.

## 2. 네이버 블로그 자동 노출

- 현재 RSS: `https://rss.blog.naver.com/refresh-bath.xml`
- `npm run sync:blog`가 최신 글 제목, 날짜, 설명, 대표 이미지를 정적 JSON으로 저장합니다.
- GitHub Actions가 매시간 RSS를 확인하고 내용이 바뀐 경우에만 `master`에 커밋합니다.
- Cloudflare Workers Builds를 `master`에 연결하면 변경된 글이 자동으로 Assets-only 정적 재배포됩니다.
- 방문 브라우저에서는 RSS API를 호출하지 않으므로 Worker 요청 한도를 사용하지 않습니다.
- 배포에는 `wrangler.assets.jsonc`를 사용하며 Worker 진입점은 포함하지 않습니다.

## 3. 구글 검색 노출

1. [Google Search Console](https://search.google.com/search-console/about)에 최종 도메인을 등록합니다.
2. DNS TXT 레코드 방식으로 소유권을 인증합니다.
3. `https://내도메인/sitemap.xml`을 제출합니다.
4. 대표 URL의 색인 생성을 요청하고 모바일 사용성, Core Web Vitals, 색인 제외 사유를 점검합니다.
5. 지역 사업자 정보가 확정되면 Google Business Profile에 상호, 서비스 지역, 홈페이지, 공식 채널을 동일하게 입력합니다.

## 4. 네이버 검색 노출

1. 네이버 서치어드바이저에 사이트를 등록하고 DNS 또는 HTML 태그로 소유권을 인증합니다.
2. 사이트맵 `/sitemap.xml`과 네이버 원본 RSS 주소를 제출합니다.
3. 사이트 설명, 상호, 서비스 지역 표현을 블로그 프로필과 일치시킵니다.

## 5. AI 검색 및 답변 노출

- `robots.txt`에서 주요 검색·AI 크롤러의 공개 페이지 접근을 허용했습니다.
- `llms.txt`, FAQ 구조화 데이터, Local Business 구조화 데이터, 명확한 서비스 지역과 시공 항목을 포함했습니다.
- 실제 주소, 전화번호, 사업자명, 운영시간이 확정되면 구조화 데이터에 추가하세요. 서로 다른 채널에서 표기가 달라지지 않게 유지하는 것이 중요합니다.
- 블로그 글은 현장 지역, 문제 증상, 선택 제품, 해결 과정, 시공 결과를 한 글 안에서 명확히 답하는 형식으로 계속 작성하세요.

## 6. 분석과 전환 측정

1. Google Analytics 4 또는 개인정보 부담이 적은 분석 도구를 선택합니다.
2. 최소 이벤트: 상담 버튼 클릭, 네이버 블로그 이동, 유튜브 이동, 인스타그램 이동, 후기 카드 클릭.
3. 쿠키 또는 광고 추적 도구를 쓰면 개인정보 처리방침과 동의 배너가 필요할 수 있습니다.

## 7. 공개 전 꼭 입력할 정보

- 최종 도메인
- 사업자 공식 상호와 대표자명
- 사업자등록번호와 통신판매 관련 정보(해당 시)
- 상담 전화번호 `010-2939-2537`의 실제 수신 가능 여부와 표기 확인
- 운영시간
- 정확한 서비스 가능 지역과 출장비 정책
- 개인정보 처리방침과 이용약관(상담 폼을 추가할 경우 필수)
